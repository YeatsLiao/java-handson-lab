# 23种设计模式交互式教学平台规划

## 1. 项目愿景 (Updated)
不仅是一个代码仓库，而是一个**沉浸式的在线互动学习平台**。
初学者通过**浏览器访问**，即可在网页上体验设计模式的动态演示。不再是枯燥的阅读代码，而是通过点击、交互、修改参数，实时观察设计模式带来的行为变化，真正实现“玩中学”。

## 2. 核心交互体验
*   **动态可视化**: 比如“单例模式”，用户点击“创建实例”按钮 10 次，屏幕上通过动画显示始终指向同一个内存块，而不是生成 10 个新方块。
*   **对比实验**: 左侧“不使用模式” vs 右侧“使用模式”。用户操作同一个业务场景（如：点击“发送通知”），直观感受代码结构和运行结果的区别。
*   **交互式图解**: 可点击的 UML 类图，高亮显示当前正在交互的代码模块。

## 3. 技术栈选型
为了实现高互动性，采用现代前端 SPA（单页应用）架构：
*   **核心框架**: **React 18 + TypeScript** (组件化开发，易于管理交互状态)
*   **构建工具**: **Vite** (极速开发体验)
*   **样式库**: **Tailwind CSS** (快速构建美观界面) + **Framer Motion** (用于制作流畅的演示动画)
*   **路由**: React Router
*   **图表/可视化**: Mermaid.js (渲染 UML) 或 React Flow

## 5. Solo 操作步骤 (开发执行SOP)
以下是你（User）可以指挥我（AI/Solo）按顺序执行的步骤：

### 第一阶段：Web 脚手架搭建
1.  **初始化工程**：使用 `vite-init` 创建 React+TS 项目。
2.  **安装依赖**：安装 Tailwind CSS, React Router, Framer Motion, Lucide React (图标)。
3.  **搭建布局**：实现响应式侧边栏布局（左侧导航，右侧内容）。

### 第二阶段：原型验证（单例模式）
请按以下指令指挥我：
1.  **"实现单例模式演示页"**：
    *   **左侧教学区**：讲解单例概念（生活类比：只有一个地球）。
    *   **右侧交互区**：
        *   设计一个“反面教材”按钮：每点击一次，屏幕上产生一个不同颜色的新球（代表新实例）。
        *   设计一个“单例模式”按钮：无论点击多少次，屏幕上始终只有同一个球在闪烁（代表获取现有实例）。
    *   **代码区**：下方展示对应的 TypeScript 代码片段。

### 第三阶段：工厂流水线
按照 **创建型 -> 结构型 -> 行为型** 的顺序，逐个开发交互页面。
*   为每个模式设计独特的交互隐喻（如：工厂模式设计一个“造车流水线”动画）。
